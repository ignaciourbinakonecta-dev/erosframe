Fix manual para shadowMap (si sigue apareciendo)
En el componente donde configuras el renderer (probablemente en tu <Canvas> o un custom renderer):tsximport * as THREE from 'three';
// ...

const { gl } = useThree();  // o en tu Canvas setup

useEffect(() => {
  gl.shadowMap.type = THREE.PCFShadowMap;  // ← este es el fix
  gl.shadowMap.enabled = true;             // si usas sombras
}, [gl]);
Si no tienes sombras activas en el maniquí (parece que no, por el fondo negro y malla blanca), puedes poner gl.shadowMap.enabled = false; para ahorrar performance y eliminar warnings relacionados.

Para el Clock (si queda algún warning residual)
Busca en tu código: new THREE.Clock() o .getDelta()
Reemplaza por:tsximport { Timer } from 'three';

const timer = new Timer();
// En useFrame:
useFrame((state, delta) => {
  timer.update();  // o usa directamente delta de useFrame (mejor)
  // ...
});
En la mayoría de casos con r3f, no necesitas crear Timer/Clock manual. Usa useFrame((state, delta) => {...}) – delta ya es correcto y no genera warnings.
